# 关键节点检查清单

> 每次任务完成后，必须检查当前项目状态是否满足以下关键节点的要求。
> 每个节点是一个独立的审查门，不通过则不得进入下一阶段。

---

## P4.1 合并前最终证据审查

- [ ] P4.0 baseline lock 存在且 SHA 匹配当前 main
- [ ] 所有 10 个 readiness gate 按 exact gate ID 验证通过
- [ ] SFT dataset 的 train/validation/heldout 三路 split 由 task/workspace family 划分（非按 step）
- [ ] 所有 trajectory 已 replay 验证，replay 结果决定接受状态（非 source label）
- [ ] rejected trajectories 已记录结构化诊断
- [ ] quarantined trajectories 未进入任何 accepted split
- [ ] forbidden_action_count = 0
- [ ] runtime_crash_count = 0
- [ ] 无 training 执行、无外部数据引入、无模型权重提交
- [ ] CI 绿色（PR head）
- [ ] readiness report 的 gate ID 与 issue 要求的 gate ID 完全一致
- [ ] 所有 teacher trajectory 记录了模型、prompt、配置、SHA、replay 和接受状态

## P4.1b 协议选择决策

- [ ] 协议选项已明确列出（continual vs independent vs 其他）
- [ ] 每个选项的 trade-off 已记录（性能影响、训练成本、回滚风险）
- [ ] 决策依据已文档化
- [ ] 决策已获得用户明确批准

## P4.2 训练授权前审查

- [ ] 数据集完整性已验证（SHA、split 规则、无泄漏）
- [ ] token audit 已完成（assistant-only loss 验证）
- [ ] smoke 结果已通过（LIMITED_SMOKE 标记明确）
- [ ] GPU 安全性已验证（4GB VRAM 约束）
- [ ] 预期训练时长已估算
- [ ] 输出路径已确认（不覆盖已有 adapter）
- [ ] 用户已明确授权训练

## P4.2 训练完成后的证据审查

- [ ] 训练日志完整（loss curve、peak GPU、无 OOM）
- [ ] adapter SHA256 与父 adapter 不同
- [ ] metrics.json 记录完整（parent SHA、trainable params、LoRA norms）
- [ ] 评估使用 frozen eval set（未修改）
- [ ] canary 样本全部失败
- [ ] 评估结果可复现（相同 model + config + eval set 可得相同结果）
- [ ] 与 P4.0 baseline 的对比数据已生成

## P4.3 能力结论与项目定位审查

- [ ] 能力提升/退化的量化证据完整
- [ ] 与 baseline 的对比覆盖所有关键指标
- [ ] 已知限制已明确文档化
- [ ] 项目定位结论有证据支撑（非模型自评）
- [ ] 后续方向建议基于实际数据（非推测）

## 任何阶段的阻断性审查

- [ ] 数据泄漏检查：train 与 frozen eval / heldout agent eval 无重叠
- [ ] silent no-op 检查：所有 action type 均已 dispatch，未知 action type 硬失败
- [ ] 工具越权检查：无 shell/network/Git push/credential/path escape 访问
- [ ] 虚假 readiness 检查：gate ID 精确匹配，非 PASS 计数
- [ ] 训练越界检查：training 仅在用户明确授权后执行
- [ ] 模型权重/secret 未提交
- [ ] frozen eval 未被静默修改
- [ ] CI 绿色（非仅本地通过）
